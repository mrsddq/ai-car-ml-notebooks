# Review Path

This repository is useful as notebook learning evidence. Its next improvement is reproducibility, not app polish.

## Notebook Review Order

1. Run the Python fundamentals notebooks.
2. Run the data exploration notebooks.
3. Run model-training notebooks from top to bottom.
4. Record assumptions, metrics, and unresolved errors in `docs/experiment-log.md`.

## Upgrade Checklist

- [ ] Add a `requirements.txt` or environment file if notebooks need non-standard packages.
- [ ] Add one summary notebook with the best experiment.
- [ ] Export one clean chart into `assets/`.
- [ ] Keep raw notebooks in place as learning history.
